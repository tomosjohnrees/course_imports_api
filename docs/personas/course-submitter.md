# Course Submitter

**Who:** A course author who has built a course repo on GitHub and wants it listed in the registry.
**Tech level:** Developer -- comfortable with GitHub, repos, and JSON file structures.
**Patience:** Medium -- willing to fix validation errors, but expects clear feedback on what went wrong.

## What they care about
- Getting their course approved and visible to learners
- Understanding exactly what the validator expects so they can fix issues on the first try
- Seeing the current status of their submissions at a glance

## How they approach the app
- Signs in, goes straight to "Submit Course," pastes a URL, and waits for validation
- Checks the dashboard to monitor pending/failed submissions
- Reads error messages carefully and cross-references with their repo structure

## What would frustrate them
- Vague validation errors that don't say which file or field failed
- No visibility into whether validation is still running or silently stuck
- Having to guess the required repo structure instead of being told upfront
